# Projekt: Dino Dash (Browser-Runner)

## Technologie-Stack
- **Framework:** Vue 3 (Composition API, `<script setup>`)
- **3D Engine:** TresJS (mit @tresjs/cientos für Loader/Helper)
- **Physik:** cannon-es
- **State Management:** Pinia
- **Sprache:** TypeScript
- **Styling:** Tailwind CSS (für das UI)

## Architektur-Prinzipien
1. **Trennung der Belange:** 
   - Die `3D-Welt` (Canvas) und das `UI-Layer` (Vue-Komponenten) sind strikt getrennt.
   - Spielstatus (Score, Coins, Game-Over-State) wird ausschließlich über **Pinia** verwaltet.
2. **Game-Loop:** 
   - Berechnungen (Physik, Kollision) finden in einem `useGameLoop` Composable statt.
   - TresJS-Komponenten reagieren auf die durch Pinia-State gesteuerten Werte.
3. **Performance:** 
   - Alle 3D-Assets liegen im `/public` Ordner (gltf/glb Format).
   - Nutzung von Objekt-Pooling für Hindernisse/Dinos, um Garbage Collection zu minimieren.

## Regeln für Claude
- **Code-Stil:** Verwende immer TypeScript. Bevorzuge modulare Composables.
- **UI:** Jede Interaktion außerhalb des Spiel-Canvas (Menüs, Scoreboard) muss als Standard-Vue-Komponente realisiert werden.
- **Kollision:** Verwende für Dinos vereinfachte `BoxCollider` (unsichtbare Boxen) statt der komplexen Meshes.
- **Logging:** Halte das Terminal sauber, nutze nur bei kritischen Fehlern `console.error`.
- **Git:** Führe Änderungen in atomaren Schritten durch. Erstelle nach jeder abgeschlossenen Spielfunktion einen Commit.
